05Root-cause capture

✗ Gap
LP knows WHICH datasets fail and HOW OFTEN — but not WHY.
The real reliability table (refreshables) carries status + counts + durations, but no error-message field. The table that would (refresh_events.error_message) is a dev fixture (104 rows, all "credentials expired (dev fixture)", 17 models) — not production.

Remediation LP-product backlog candidate

The Fabric Refreshables API returns a serviceExceptionJson / error payload on failed refreshes. Capturing it onto refreshables (or populating refresh_events from real data) would turn this diagnostic from "667 datasets are failing" into "667 datasets are failing because of these 3 error classes" — the difference between a symptom and a work order.
