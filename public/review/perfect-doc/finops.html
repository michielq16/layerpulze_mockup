05Refresh cost analysis

✓ Live

Refresh operations consume €2,120/mo — 44% of the model's bill. One table (FactTelemetry) accounts for 32% of that subtotal in a single refresh window. Switching that one table to incremental refresh is the single highest-impact technical lever in this report.

Why FactTelemetry costs so much

FactTelemetry is a full-refresh table at 8.1 GB. Every refresh re-reads the entire table from the source Lakehouse and rebuilds VertiPaq compression — even though only the last day's worth of data has changed. Incremental refresh, partitioned by day, would refresh the last 1–2 daily partitions instead of all 730. LayerPulse BPA rule BPA-INC-001 identified this as a high-confidence optimization.

Estimated impact

Switching FactTelemetry to daily incremental refresh: refresh time drops from 44 min to ~18 min, refresh cost drops from €680/mo to ~€280/mo (−€400/mo, −59%). Implementation: ~2 days BI Engineering effort; LayerPulse provides the PartitionByYear template script.
