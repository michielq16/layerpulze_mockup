The one-sentence version

One page, three admin jobs

Three different admin jobs share one user (the admin), so they share one page — split by tab so each has a clean home and the page doesn't become a wall.

The "planned ingestion axes" teaser came off (the roadmap belongs in the backlog, not here), the status visual switched from squares to dots (it's a 1-bit health signal, not a heatmap), and the cron-arms list was re-audited against the LayerPulse code so it actually matches what's running.

The three tabs

Connection · Ingestion · Pricing

Attach the tenant.

  • Danger zone (delete environment — two-step)

♻️ Ingestion

Operate the cron.

  • Sync toggle · Schedule · Timezone

💰 Pricing

Money truth-of-record.

  • Capacity €/mo per F-SKU

  • Effective-date history
  • Powers share-of-bill math everywhere else
The cron arms — re-audited against the LayerPulse repo

The mockup said "Five canonical arms." LayerPulse actually runs six. Reports & Apps graduated from "planned" to "shipped" and was added; Model introspection runs as the prerequisite that the others fan out over; FUAM is deprecated and intentionally omitted.

What changed in V2

Three deliberate edits

1. Squares → dots. The 30-run completion grid + per-arm grids render circular dots now — matches the rest of the product's status vocabulary, removes the "mini-heatmap" misread.

2. Removed the "Planned ingestion axes" card. Some entries were already shipped (Reports & Apps); the card was drifting and the roadmap belongs in BACKLOG.md, not on the admin page.

3. Re-audited cron arms against the LayerPulse repo. Six active arms instead of five (Reports & Apps added; FUAM correctly omitted as deprecated). Each arm's meta line now matches the actual API path / code pattern.
